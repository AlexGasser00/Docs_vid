Виджет навигации

Руководство по конфигурированию

1. Базовая настройка и источники данных

Виджет навигации показывает план объекта, следит за авариями и температурой, а также помогает быстро переключаться между связанными дашбордами.

Добавление виджета на дашборд

  1. Откройте нужный дашборд в Смарт платформе.
  2. В правом верхнем углу нажмите кнопку «Редактировать дашборд» (иконка карандаша).
  3. Нажмите кнопку «Добавить виджет» в центре экрана.
  4. Найдите набор виджетов «Навигация» и выберите виджет «Навигация здания v8». После этого откроется окно его настройки.

Настройка источников данных

Чтобы виджет вовремя реагировал на аварии и обновлял температуру, подключите к нему телеметрию.

Как это сделать:

  1. В блоке «Источники данных» нажмите «Добавить» (Add).
  2. «Тип источника» — выберите подходящий вариант, обычно это «Устройство».
  3. «Устройство» — укажите устройство, с которого виджет будет получать телеметрию.
  4. «Ключи данных» — впишите сюда все параметры телеметрии, которые понадобятся виджету (температура, общие аварии, аварии для конкретных кнопок).

Базовые настройки

Теперь, когда данные подключены, перейдите на вкладку «Внешний вид». В самом верху вы найдете базовые параметры:

Название в интерфейсе Описание и настройка
Базовый URL Корневой адрес вашего стенда.

Важный момент

Если виджет работает на том же стенде, где вы его настраиваете, просто оставьте поле пустым. Заполняйте его, только если виджет должен ссылаться на дашборды с другого сервера.
Подложка Фоновое изображение объекта, поверх которого отображаются кнопки навигации и интерактивные зоны.

Кликните на поле загрузки и выберите картинку из галереи Смарт Платформы (Используйте PNG, JPG или WebP). Подложка сама подстроится под размеры виджета без потери пропорций. При необходимости уменьшения веса или формата файла, воспользуйтесь сервисом Squoosh.app. Загрузите подготовленную версию в галерею Смарт Платформы.

2. Настройка кнопок навигации

Виджет сам генерирует кнопки для перехода к инженерным системам или корпусам. Вы управляете их списком, порядком и настройками через один JSON-массив.

Формат конфигурации

Каждая кнопка — это отдельный блок (объект) внутри массива. Вот так выглядит настройка для двух кнопок:

[
  {
    "id": 1,
    "name": "СберСити",
    "dashboardId": "54a51780-0279-11f1-ad6c-2fa7a6b600a9",
    "column": "main",
    "orderInColumn": 0,
    "iconType": "apartment",
    "alarmKey": ""
  },
  {
    "id": 2,
    "name": "Квартиры \n(корпус 1)",
    "dashboardId": "268ce410-2784-11f1-9f64-c787c2280ca6",
    "column": "main",
    "orderInColumn": 1,
    "iconType": "apartment",
    "alarmKey": "alarms_apartments_1"
  }
]

Описание параметров

Параметр Тип данных Описание
id Целое число Уникальный номер кнопки. Нужен виджету, чтобы правильно привязать к ней аварии. Убедитесь, что номера не повторяются.
name Строка Название, которое появится на кнопке. Чтобы перенести текст на новую строку, вставьте \n.
dashboardId Строка Уникальный идентификатор (UUID) дашборда, который откроется по клику. Если оставить поле пустым, кнопка станет неактивной.

Что бы найти UUID откройте нужный целевой дашборд. В адресной строке браузера вы увидите ссылку вроде https://ваш-домен.ru/dashboards/54a51780-0279-11f1-ad6c-2fa7a6b600a9. Длинный код сразу после /dashboards/ — это и есть нужный UUID.
column Строка Идентификатор колонки. В текущей версии виджета всегда пишите "main".
orderInColumn Целое число Позиция кнопки в списке сверху вниз. 0 — самая верхняя, 1 — под ней и так далее. Если нужно поменять кнопки местами, просто измените эти индексы.
iconType Строка Название векторной иконки для кнопки (список доступных вариантов смотрите в разделе "Генератор кнопок навигации").
alarmKey Строка Ключ телеметрии, по которому кнопка будет отслеживать аварии.

3. Информационная панель и графики

            
Информационная панель (блок в правом верхнем углу) нужна для быстрого обзора показателей уличной температуры и количества активных аварий на объекте. По клику на панель открывается всплывающее окно с графиком температуры.

            
            
Параметры этого блока находятся в разделе «НАСТРОЙКИ ИНФОРМАЦИОННОЙ ПАНЕЛИ»:


            

                

                    Название в интерфейсе
                    В чем измеряется
                    Описание
                

                

                    Заголовок информационной панели
                    Текст
                    Текст слева от значения температуры (по умолчанию — «Уличная температура»). Он же служит кликабельной кнопкой для открытия графика.
                

                

                    Ширина панели
                    px
                    Ширина всего блока. Если поставить 0, панель сама подстроится под размер текста.
                

                

                    Высота панели
                    px
                    Высота информационной панели.
                

                

                    Размер шрифта заголовка
                    px
                    Размер букв для текста из поля «Заголовок информационной панели».
                

                

                    Размер шрифта температуры
                    px
                    Размер цифр, показывающих температуру.
                

                

                    Размер шрифта количества аварий
                    px
                    Размер цифры счетчика аварий в правой части.
                

                

                    Размер шрифта подписи 'аварий'
                    px
                    Размер слова «аварий» рядом со счетчиком.
                

                

                    Ключ температуры
                    Текст (Key)
                    Ключ телеметрии, откуда виджет берет данные о температуре (например, temperature_street_today).
                

                

                    Ключ общих аварий
                    Текст (Key)
                    Ключ телеметрии для счетчика аварий (например, alarms_count).
                

                

                    ID состояния для графика температуры
                    Текст (ID)
                    Техническое имя состояния (State ID), которое откроет окно с графиком при клике.
                

            


            
Настройка мониторинга аварий и температуры

            
Панель наглядно сигнализирует об инцидентах:

            
    
                
  • Пока аварий нет (значение ключа равно 0), панель сохраняет стандартный вид.
    • 
                
  • При появлении хотя бы одной аварии, иконка и цифра счетчика окрашиваются в красный цвет.
    • 
            

            
Чтобы это работало, пропишите ключи данных для температуры и аварий в настройках виджета.

            
            

                Важное правило
                
Перед тем как вписать ключи, убедитесь, что они добавлены во вкладку «Данные» (См. раздел "Базовая настройка"). Если виджет не найдет ключ в источнике данных, он ничего не покажет.

            


            
Настройка температурного графика

            
Для открытия подробного графика температуры по нажатию на панель, нужно создать в дашборде новое «состояние» (State):

            
            
    
                
  1. Перейдите в режим редактирования дашборда. Нажмите кнопку «Состояния» (иконка слоев на верхней панели).
    2. 
                
  2. Добавьте новое состояние (кнопка +) и заполните поля:
                    
      
                        
    • "Имя" - любое понятное название, например «График температуры на улице».
      • 
                        
    • "ID состояния (State ID)" - короткий идентификатор, например temp_chart.
      • 
                    
    
                
    3. 
                
  3. Сохраните изменения. Теперь вернитесь в настройки виджета навигации и впишите ваш temp_chart в поле «ID состояния для графика температуры».
    4. 
                
  4. Теперь кликните на информационную панель. Откроется пустое диалоговое окно. Нажмите «Добавить новый виджет».
    5. 
                
  5. Найдите категорию «Charts» и выберите «Line chart».
    6. 
                
  6. Настройте график: укажите устройство с уличной температурой, выберите "Использовать временной промежуток виджета" и установите временной интерва (например, 30 дней).
    7. 
                
  7. Растяните график на всё окно и сохраните дашборд.
    8.

4. Настройка интерактивных зон (Конфигуратор)

            
Интерактивные зоны позволяют превратить обычное изображение подложки в динамическую карту. При наведении зона подсвечивается, а при клике — осуществляется переход на смежный дашборд.


            
Формат конфигурации зон (JSON)
	
            
Каждая зона — это отдельный блок в JSON-массиве. Найти этот массив можно в поле «Конфигурация зон» (раздел «ИНТЕРАКТИВНЫЕ ЗОНЫ»). Так выглядит код для одной зоны:


            


[
  {
    "id": "zone_1778763382727_8jkm",
    "name": "Секция 1",
    "dashboardId": "268ce410-2784-11f1-9f64-c787c2280ca6",
    "points": [
      { "x": 249, "y": 718 },
      { "x": 249, "y": 424 },
      { "x": 286, "y": 416 },
      { "x": 292, "y": 382 },
      { "x": 317, "y": 371 },
      { "x": 333, "y": 320 },
      { "x": 359, "y": 318 },
      { "x": 364, "y": 285 },
      { "x": 467, "y": 263 },
      { "x": 538, "y": 276 },
      { "x": 571, "y": 245 },
      { "x": 661, "y": 239 },
      { "x": 712, "y": 254 },
      { "x": 711, "y": 656 },
      { "x": 428, "y": 768 }
    ],
    "color": "#f0ff24",
    "originalWidth": 2112,
    "originalHeight": 1208
  }
]


            


            
Описание параметров

            

                

                    Параметр
                    Тип данных
                    Описание
                

                

                    id
                    Строка
                    Уникальный номер зоны. Конфигуратор создает его сам (например, zone_1778763382727_8jkm). Менять его вручную не нужно, главное — чтобы он не повторялся у других зон.
                

                

                    name
                    Строка
                    Название зоны. Пользователь увидит его во всплывающей подсказке при наведении курсора.
                

                

                    dashboardId
                    Строка
                    UUID дашборда для перехода по клику. Если оставить пустым, зона будет подсвечиваться, но переход на смежный дашборд осуществлен не будет.

(Что такое UUID и как его найти смотри в разделе 2).		
                

                

                    points
                    Массив объектов
                    Координаты x и y для точек, по которым строится контур зоны. Координаты считаются в пикселях от исходного размера картинки.

 Минимальное количество точек для создания зоны - 3 точки. Последняя точка автоматически соединится с первой.		
                

                

                    color
                    Строка (HEX)
                    Цвет подсветки при наведении. Выбирается на палтире внутри конфигуратора в формате HEX (например, #f0ff24 — жёлтый).
                

                

                    originalWidth
                    Целое число
                    Ширина исходной подложки на которой вы создаете зону. Опираясь на это значение, виджет понимает, как масштабировать зону, если изменится размер окна виджета.
                

                

                    originalHeight
                    Целое число
                    Высота исходной картинки. Работает в паре с originalWidth.
                

            


            

                Как работает масштабирование зон
                
Виджет автоматически подстраивает нарисованные зоны под размер виджета. Зоны всегда останутся привязанными к своим объектам.

                
ВАЖНО! При наличии сконфигурированных зон, изменение подложки может привести к смещению зон. Отрисуйте зоны снова или поправьте готовую конфигурацию в ручном режиме.

            


            
Создание зон через конфигуратор

            
Прежде чем начать, перейдите в настройки виджета (раздел «КОНФИГУРИРОВАНИЕ ЗОН») и включите две опции:

            
    
                
  • «Включить интерактивные этажи» — для запуска функционала.
    • 
                
  • «Режим редактирования этажей» — для доступа к кнопке запуска конфигуратора.
    • 
            

            
Примените настройки.

            
Как нарисовать зону:

            
    
                
  1. Кликните по иконке карандаша в правом нижнем углу виджета. Кнопка станет синей, и откроется панель конфигуратора.
    2. 
                
  2. Впишите название зоны, вставьте UUID дашборда и выберите цвет подсветки будущей зоны.
    3. 
                
  3. Нажмите «✏️ Начать рисовать». Кликайте мышью по границам объекта на плане. Каждое нажатие — новая точка.
    4. 
                
  4. Что бы удалить крайнюю выставленную точку, нажмите «↩️ Отмена».
    5. 
                
  5. Когда контур готов, нажмите «💾 Сохранить». Зона появится в списке и на подложке.
    6. 
                
  6. Выйдите из редактора (снова нажмите на карандаш), а затем сохраните дашборд.
    7. 
                
  7. После завершения конфигурирования, отключите опцию «Режим редактирования зон» в настройках виджета.
    8. 
            


            
Управление списком зон

            
Внизу панели конфигуратора находится список всех созданных зон. Здесь вы можете:

            
    
                
  • «Редактировать» (кнопка «Ред.») — на плане появятся точки зоны. Взаимодейтсвуйте с точками мышью, чтобы поправить контур.
    • 
                
  • «Удалить» (кнопка «Уд.») — полностью стирает сконфигурированную зону.
    • 
                
  • «Экспорт JSON» — Копирует код сконфигурированных зон. Удобно, если нужно перенести настройки на другой стенд или сделать бэкап.
    • 
            


            
Ручное редактирование зон

            
В некоторых случаях быстрее править код напрямую, чем изменять зоны в редакторе. Для этого найдите поле «Конфигурация зон» в настройках виджета. Ручная правка полезна, когда нужно:

            
    
                
  • Быстро скопировать зоны с одного дашборда на другой.
    • 
                
  • Массово заменить UUID или цвета (через поиск/замену).
    • 
                
  • Восстановить работу из бэкапа.
    • 
                
  • Скорректировать точки, если зоны сместились.
    • 
            


            

                Правила ручной правки
                
    
                    
  • Следите за скобками — это должен быть валидный JSON-массив.
    • 
                    
  • У каждой зоны обязательно должны быть поля: id, name, points, color, originalWidth, originalHeight.
    • 
                    
  • Если переход не нужен, оставьте поле dashboardId пустым ("").
    •
